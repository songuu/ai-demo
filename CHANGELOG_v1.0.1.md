## 🔧 Bug 修复

### 1. Claude Code "No messages returned" 错误处理 ✅

**问题描述**:
```
Error: No messages returned
at nD9 (file:///C:/App/nvm/v24.11.1/node_modules/@anthropic-ai/claude-code/cli.js:4886:73)
```

**根本原因**:
- 工作目录不正确或不存在
- Claude Code 在错误的目录中寻找会话历史
- 缺少详细的错误提示

**解决方案**:

#### 1.1 增强工作目录验证
```dart
// 在 cod_cli_runner.dart
if (session.provider.toLowerCase() == 'claude') {
  if (workingDir == null || workingDir.isEmpty) {
    // 自动回退策略
    // 1. 尝试日志文件目录
    // 2. 最后使用当前目录
    // 添加详细警告日志
  }
}
```

#### 1.2 详细的警告信息
```
[warning] Claude Code requires a working directory. Using: <directory>
[info] If you see "No messages returned" error, make sure you are in the correct project directory.
```

#### 1.3 改进的错误提示
- 日志文件中记录详细信息
- 帮助用户理解问题
- 提供解决建议

**代码更改**:
- `cod_cli_runner.dart`: 增强 `_resolveWorkingDirectory()` 方法
- `cod_cli_runner.dart`: 添加 Claude Code 特殊处理
- `cod_cli_runner.dart`: 增加警告日志

## 📝 使用建议

### 1. 避免 "No messages returned" 错误

**最佳实践**:
```
✅ 使用应用的 "Resume" 按钮
✅ 使用 "Terminal" 下拉菜单
✅ 使用 "Copy Cmd" 复制完整命令

❌ 不要手动在错误的目录运行
❌ 不要忽略工作目录
```

### 2. 管理会话

**推荐流程**:
```
1. 定期导入历史会话
2. 使用搜索和过滤功能
3. 删除不需要的会话
4. 保持会话列表整洁
```
