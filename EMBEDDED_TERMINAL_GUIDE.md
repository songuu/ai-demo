# 内置终端使用指南

## 🎯 概述

内置终端是一个集成在应用中的交互式终端，允许你直接在应用内运行和续聊 CLI 会话（Claude Code、Codex、Gemini），无需切换到外部终端。

---

## ✨ 主要功能

### 1. 自动 Resume 会话 ✅
点击 "Resume" 按钮时，应用会：
- ✅ 自动切换到内置终端视图
- ✅ 自动执行 resume 命令
- ✅ 启动交互式会话
- ✅ 准备接收你的输入

### 2. 交互式对话 ✅
- ✅ 在输入框中输入消息
- ✅ 按 Enter 或点击发送按钮
- ✅ 实时查看 CLI 响应
- ✅ 支持多轮对话

### 3. 命令执行 ✅
- ✅ 运行任意命令
- ✅ 内置命令支持（cd, pwd, clear, help）
- ✅ 进程管理（启动、停止）
- ✅ 输出复制

---

## 🚀 使用方法

### 方式 1: Resume 会话（推荐）

**步骤**:
```
1. 在会话列表中选择要恢复的会话
2. 点击右上角的 "Resume" 按钮
3. 应用自动切换到内置终端
4. 等待几秒钟，看到 "Interactive session started" 提示
5. 在底部输入框输入你的消息
6. 按 Enter 开始对话！
```

**预期效果**:
```
Terminal initialized
Working directory: E:\project
Auto-resuming: cd E:\project && claude --continue
> cd E:\project && claude --continue
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ Interactive session started successfully!
✓ You can now type your messages in the input field below
✓ Press Enter or click Send button to submit
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[Claude Code 的输出...]
```

### 方式 2: 手动切换到终端视图

**步骤**:
```
1. 选择一个会话
2. 点击右上角的 "Terminal" 按钮
3. 应用切换到内置终端视图
4. 点击顶部的播放按钮 ▶ 启动会话
5. 开始对话
```

### 方式 3: 直接在详情页切换

**步骤**:
```
1. 选择会话后，默认显示对话历史
2. 点击 "Terminal" 按钮切换到终端视图
3. 终端自动启动
4. 开始输入
```

## 💡 交互技巧

### 输入消息

**正确方式**:
```
1. 等待 "Interactive session started" 提示出现
2. 确保状态显示为 "Running..."（绿色）
3. 在底部输入框输入消息
4. 按 Enter 或点击发送按钮 📤
```

**提示文字含义**:
- `Type your message and press Enter...` - 进程正在运行，可以输入消息
- `Type command (or "help" for available commands)...` - 进程未运行，可以输入命令

### 多轮对话

**示例**:
```
你: 帮我写一个 Python 函数计算斐波那契数列
[等待 Claude 响应...]
Claude: [提供代码...]

你: 请添加单元测试
[等待 Claude 响应...]
Claude: [提供测试代码...]
```

### 内置命令

当进程未运行时，可以使用内置命令：

| 命令 | 说明 | 示例 |
|-----|------|-----|
| `cd <dir>` | 切换目录 | `cd E:\project` |
| `pwd` | 显示当前目录 | `pwd` |
| `clear` / `cls` | 清空输出 | `clear` |
| `help` | 显示帮助 | `help` |
| 其他 | 执行任意命令 | `node --version` |

## 🎨 UI 元素说明

### 顶部标题栏

```
🔴 🟡 🟢  Terminal - [会话标题]  [Running...]  📋 📄 ⏹️ ▶️
```

**图标功能**:
- 🔴 - 关闭终端视图
- 🟡 - （保留）
- 🟢 - （保留）
- 📋 - 复制命令
- 📄 - 复制输出
- ⏹️ - 停止进程（运行时显示）
- ▶️ - 启动进程（停止时显示）

**状态指示**:
- `Running...`（绿色） - 进程正在运行，可以交互
- `Completed`（灰色） - 进程已完成
- `Failed`（红色） - 进程失败

### 输出区域

显示所有输出，带颜色区分：

| 颜色 | 类型 | 说明 |
|-----|------|-----|
| 灰色 | 系统消息 | `Terminal initialized` |
| 白色 | 标准输出 | CLI 的正常输出 |
| 红色 | 错误输出 | 错误和警告信息 |
| 绿色加粗 | 用户输入 | `> 你的命令` |

### 底部输入区

```
$ [输入框...................................] 📤
```

- `$` - 命令提示符
- 输入框 - 输入消息或命令
- 📤 - 发送按钮

## ⚙️ 工作原理

### 进程管理

```
┌─────────────────────────────────────────┐
│         点击 Resume 按钮                │
└─────────────┬───────────────────────────┘
              │
      ┌───────▼────────┐
      │ 切换到终端视图  │
      └───────┬────────┘
              │
      ┌───────▼────────┐
      │ 延迟 500ms      │
      │ (等待 UI 渲染)  │
      └───────┬────────┘
              │
      ┌───────▼────────────────┐
      │ 自动执行 _runCommand() │
      └───────┬────────────────┘
              │
      ┌───────▼───────────────────┐
      │ Process.start()           │
      │ - 设置工作目录            │
      │ - 配置环境变量            │
      │ - 连接 stdin/stdout/stderr│
      └───────┬───────────────────┘
              │
      ┌───────▼────────┐
      │ 进程运行中      │
      │ _isRunning=true │
      └───────┬────────┘
              │
      ┌───────▼───────────┐
      │ 监听输出流        │
      │ - stdout → 白色   │
      │ - stderr → 红色   │
      └───────┬───────────┘
              │
      ┌───────▼────────────────┐
      │ 等待用户输入           │
      │ 输入框显示提示文字     │
      └───────┬────────────────┘
              │
      ┌───────▼────────────────┐
      │ 用户输入并按 Enter      │
      └───────┬────────────────┘
              │
      ┌───────▼─────────────────┐
      │ _sendInput()            │
      │ - process.stdin.write() │
      │ - 显示为绿色            │
      └───────┬─────────────────┘
              │
      ┌───────▼────────────┐
      │ CLI 处理并响应     │
      │ stdout 显示响应    │
      └───────┬────────────┘
              │
              ▼
      [循环：继续接收输入]
```

### 输入处理逻辑

```dart
if (_process != null && _isRunning) {
  // 进程正在运行 → 发送到 stdin
  _process.stdin.write('$input\n');
  _process.stdin.flush();
} else {
  // 进程未运行 → 处理内置命令或启动新命令
  if (input.startsWith('cd ')) { ... }
  else { _runCommand(input); }
}
```

## 🔧 故障排除

### 问题 1: 输入没有响应

**症状**:
- 输入消息后按 Enter
- 没有任何反应
- 看不到输出

**原因**:
- 进程未启动
- 进程已退出
- CLI 工具未正确安装

**解决方案**:

1. **检查进程状态**
   ```
   查看顶部状态栏：
   - 如果显示 "Completed" 或 "Failed" → 进程已停止
   - 需要点击 ▶️ 按钮重新启动
   ```

2. **查看输出**
   ```
   检查输出区域是否有错误消息（红色）
   例如：
   - "No such file or directory" → CLI 工具未安装
   - "No messages returned" → 工作目录不正确
   ```

3. **手动重启**
   ```
   1. 点击 ▶️ 播放按钮
   2. 等待 "Interactive session started" 提示
   3. 重新输入消息
   ```

### 问题 2: "Interactive session started" 未出现

**症状**:
- 点击 Resume 后只看到：
  ```
  Terminal initialized
  Working directory: E:\project
  Auto-resuming: cd E:\project && claude --continue
  > cd E:\project && claude --continue
  ```
- 没有后续提示

**原因**:
- CLI 命令未被识别为交互式
- 工作目录不正确
- CLI 工具未安装

**解决方案**:

1. **检查 CLI 工具**
   ```bash
   # 在外部终端测试
   claude --version
   ```

2. **检查工作目录**
   ```
   查看输出的 "Working directory"
   确保该目录存在且有会话历史
   ```

3. **查看错误输出**
   ```
   等待几秒，看是否有红色错误消息
   ```

### 问题 3: 输出显示不完整

**症状**:
- 只看到部分输出
- 输出被截断

**原因**:
- 输出更新延迟
- UI 未自动滚动

**解决方案**:

1. **手动滚动**
   ```
   滚动输出区域到底部
   ```

2. **等待加载**
   ```
   大量输出可能需要几秒才能完全显示
   ```

3. **复制输出查看**
   ```
   点击 📄 按钮复制所有输出
   粘贴到文本编辑器查看完整内容
   ```

### 问题 4: 进程意外退出

**症状**:
- 对话中途状态变为 "Completed" 或 "Failed"
- 无法继续输入

**原因**:
- CLI 工具崩溃
- 工作目录被删除
- 网络问题（API 调用失败）

**解决方案**:

1. **查看退出码**
   ```
   输出区域会显示：
   Process exited with code: <exit_code>
   
   - 0: 正常退出
   - 非0: 错误退出
   ```

2. **查看错误消息**
   ```
   检查退出前的红色错误输出
   ```

3. **重新启动**
   ```
   点击 ▶️ 按钮重新启动会话
   ```

## 🆚 内置终端 vs 外部终端

| 特性 | 内置终端 | 外部终端 |
|-----|----------|----------|
| **集成度** | ✅ 完全集成 | ⚠️ 需切换窗口 |
| **自动启动** | ✅ 一键 Resume | ⚠️ 手动输入命令 |
| **输出保存** | ✅ 自动保存 | ❌ 需手动复制 |
| **历史查看** | ✅ 随时切换 | ❌ 终端关闭即丢失 |
| **多会话** | ✅ 切换方便 | ⚠️ 需多个窗口 |
| **功能完整性** | ⚠️ 基础功能 | ✅ 完整终端功能 |
| **性能** | ⚠️ 大量输出可能慢 | ✅ 原生性能 |

**推荐使用场景**:

**内置终端** ✅:
- 快速 Resume 会话
- 短对话和测试
- 需要频繁切换会话
- 需要对比历史对话

**外部终端** ✅:
- 长时间对话
- 大量输出（如代码审查）
- 需要完整终端功能
- 需要使用其他终端工具

## 💡 最佳实践

### 1. Resume 会话

✅ **推荐流程**:
```
1. 点击 "Resume" 按钮
2. 等待 3-5 秒直到看到 "Interactive session started"
3. 开始输入消息
4. 观察输出和状态
```

### 2. 处理长对话

✅ **建议**:
```
1. 每个问题单独发送
2. 等待响应完成后再发送下一个
3. 定期复制输出保存
4. 如果需要大量交互，考虑使用外部终端
```

### 3. 错误恢复

✅ **步骤**:
```
1. 查看错误消息
2. 理解问题原因
3. 修复问题（如安装工具、修正目录）
4. 点击 ▶️ 重新启动
```

### 4. 输出管理

✅ **技巧**:
```
1. 使用 clear 清空旧输出
2. 定期复制重要输出
3. 切换到对话视图查看历史
4. 使用搜索过滤对话
```

## 🎓 高级技巧

### 1. 快捷键

- `Enter` - 发送输入
- `Ctrl+C` - （保留）复制
- `Ctrl+V` - 粘贴

### 2. 命令历史

内置终端当前不支持命令历史（↑↓），但你可以：
- 复制之前的命令重新粘贴
- 使用对话历史查看之前的交互

### 3. 多会话管理

```
1. 在会话列表选择第一个会话
2. Resume 并进行对话
3. 切换到另一个会话
4. Resume 第二个会话
5. 在两个会话间快速切换
```

### 4. 输出解析

输出中的特殊标记：
- `[info]` - 信息
- `[warning]` - 警告
- `[error]` - 错误
- `>` - 你的输入
